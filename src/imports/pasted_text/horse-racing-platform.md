Создать веб-сайт, который объединит:
1.	Спортивный портал — афиша скачек, результаты, статистика побед.
2.	Племенной учет — родословные, случной сезон, рождение жеребят.
3.	Торговая площадка — продажа и бронирование жеребят с прогнозом их будущей стоимости.
4.	Личные кабинеты — для владельцев, тренеров, жокеев, ветеринаров.

3. Типы пользователей и их права
3.1. Гость (незарегистрированный)
Что видит:
•	Афишу предстоящих скачек.
•	Результаты прошедших забегов.
•	Каталог лошадей на продажу (базовая информация, без цен и контактов).
•	Общую информацию о лошадях-звездах.
Чего НЕ видит: личные данные, контакты владельцев, точные цены.

3.2. Владелец лошади (частный)
Что может делать:
•	Видеть список СВОИХ лошадей.
•	Смотреть полную карточку каждой лошади (родословная, история выступлений, медкарта).
•	Получать уведомления о ближайших стартах своих лошадей.
•	Просматривать финансовую статистику по лошади (сколько выиграно, сколько потрачено на содержание).
•	Общаться с тренером через внутренние сообщения.

3.3. Конный завод / Владелец завода
Что может делать:
•	Вести полный учет всех лошадей завода.
•	Добавлять новых лошадей в систему.
•	Управлять случным сезоном (планировать вязки).
•	Добавлять жеребят в базу (с привязкой к родителям).
•	Выставлять жеребят на продажу (устанавливать цену, статус).
•	Подавать заявки на участие лошадей в скачках.

3.4. Тренер
Что может делать:
•	Видеть всех лошадей, которых тренирует.
•	Вести журнал тренировок (дата, вид нагрузки, самочувствие лошади).
•	Подавать заявки на скачки от имени владельца.
•	Получать отчеты от жокеев после заездов.
•	Видеть ветеринарные ограничения по лошадям.

3.5. Жокей
Что может делать:
Видеть тактическую информацию перед заездом (особенности лошади, рекомендации тренера).
После заезда заполнять электронный отчет:
o	Поведение лошади на старте, дистанции, финише.
o	Самочувствие лошади.
o	Замечания по снаряжению.
o	Рекомендации тренеру.
Отправлять отчет тренеру через систему.

3.6. Ветеринарный врач
Что может делать:
•	Вести электронную медицинскую карту на каждую лошадь:
o	Прививки (с напоминаниями о следующих).
o	Заболевания и травмы.
o	Лечение и назначения.
o	Результаты анализов (возможность прикрепить файл).
•	Вносить ограничения для тренировок (например: «исключить прыжки на 2 недели»).
•	Получать уведомления о плановых обработках.

3.7. Администратор системы
Что может делать:
•	Управлять пользователями (создавать, блокировать, менять роли).
•	Создавать события (скачки) в календаре.
•	Подтверждать заявки на участие в скачках.
•	Вносить официальные результаты забегов.
•	Настраивать права доступа.
•	Смотреть логи действий пользователей.

4. Функциональные модули (что должно работать)
Модуль 1. Учет лошадей (цифровая карточка)
Что храним на каждую лошадь:
•	Кличка, пол, масть, год рождения, страна рождения.
•	Заводчик, владелец (может меняться).
•	Фотографии (несколько штук).
•	Родословная (отец, мать, деды, прадеды) — с возможностью кликнуть и перейти в их карточку.
•	Статус: «в тренинге», «на отдыхе», «в случке», «продан», «выбыла».
Спортивная карьера (автоматически):
Таблица всех забегов: дата, ипподром, дистанция, место, выигранная сумма.
Итоговая статистика: Всего стартов: 24, Побед: 8, Сумма выигрыша: 3 450 000 руб.
Медицинский блок (для ветврача):
•	История прививок.
•	История заболеваний и лечения.
•	График обработок.

Модуль 2. Скачки и результаты
Календарь скачек:
•	Список предстоящих забегов (дата, ипподром, дистанция, призовой фонд, статус приема заявок).
•	Возможность фильтровать по дате, ипподрому.
Подача заявок:
•	Тренер/владелец выбирает лошадь и забег → отправляет заявку.
•	Администратор видит заявку → подтверждает или отклоняет.
•	Участники забега публикуются в карточке события.
Результаты:
•	Администратор вносит результаты (1-е место, 2-е место, 3-е место, остальные с временем).
Система автоматически:
o	Добавляет выигранную сумму в статистику лошади.
o	Обновляет рейтинги.
o	Отправляет уведомление владельцу.

Модуль 3. Случной сезон и разведение
План случек:
•	Создание графика: кобыла + жеребец + планируемая дата вязки.
•	Статусы: «запланировано», «проведено», «жеребость подтверждена», «жеребость не подтверждена».
Учет жеребят:
•	При рождении жеребенка создается его карточка с автоматической привязкой к родителям.
•	Поля: дата рождения, пол, масть, примечания.
•	Статус: «в заводе», «на продаже», «забронирован», «продан».
Бронирование:
•	Покупатель оставляет заявку на жеребенка (из каталога).
•	Владелец завода видит заявку, может связаться с покупателем.
•	Статус жеребенка меняется на «забронирован».

Модуль 4. Продажи и прогнозирование
Каталог на продажу:
•	Список лошадей/жеребят с ценой, фото, родословной.
•	Фильтры: по возрасту, по полу, по родителям, по цене.
•	Возможность для зарегистрированных пользователей оставлять заявки.
Прогноз стоимости жеребенка (важная функция!):
•	Система анализирует:
o	Сколько выиграл отец за карьеру.
o	Сколько выиграла мать (или ее близкие родственники).
o	По какой цене продавались братья и сестры (от тех же родителей).
•	На основе этих данных выводится прогнозируемая цена.
Пример: «Отец выиграл 5 забегов (сумма 2 млн руб.), мать — 2 забега (800 тыс. руб.), старший брат продан за 1.2 млн руб. → прогноз стоимости: 1.1 — 1.5 млн руб.»

Модуль 5. Аналитика и отчеты
Для владельца:
•	Доходы/расходы по каждой лошади.
•	Динамика результатов (график мест/выигрышей по годам).
Для тренера:
•	Анализ эффективности тренировок.
•	Сравнение показателей лошадей.
Для администратора:
•	Общая статистика по заводу/ипподрому.
•	Отчеты по продажам, случкам, выступлениям.
